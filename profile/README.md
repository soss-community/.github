## What we build

We build **precision tools**, not Swiss Army knives.

Every package, module and software solution we release has a clear purpose. We do not add features for the sake of features. Security is part of the architecture from day one, not an afterthought.

What we provide:

- **Modules** focused, reusable building blocks
- **Packages** structured collections ready to integrate
- **Software** complete solutions for specific use cases

Where it makes sense, we build variants. A small-to-mid usage version and an enterprise variant  
(e.g. async for lightweight use, multithreading/service architecture for scale) separate tools, not bloated toggles.

Our starting point:  
tools that help us build faster, cleaner and more intentionally. 
Turning ideas and architectures and into working software.

## Licenses

We do not adjust our licenses for anyone. Not for requests. Not for money.


### Modules & Packages
Usage requires credit, a mention in your credits, documentation or README that this module/package was used.  
That is all.

→ [View Module/Package License](/licenses/module_license)

### Software
- **Private use**: Use and modify freely for yourself.
- **Business use**: Use as-is. Modification is not permitted. Why? We build with security in mind. We cannot guarantee the integrity of modified versions used in production environments.
- **Interfaces**: If you need a connection we have not built yet, feel free to reach out to us. If it makes sense, we will build it with proper auth, documentation and clear scope of what can be queried.

→ [View Software License](/licenses/software_license)

We do not ship ads. Ever. Ads are a security risk. Full stop.

---

## Security

Security issues are fixed immediately. No scheduled patch cycles.

Versions with known security vulnerabilities are removed from distribution as soon as possible. We do not leave broken versions available.

## Contributing

We do not accept pull requests.

We do accept issues, including ones with code examples. If your idea or code influences something we build, we will credit you.  
Even if it came up in a conversation at a meetup.  
We are human, things slip through, if they do, just reach out.

→ [How to become a community member](/community/README.md)
